JS程序设计-类型系统

类型系统



  • 标准类型:(6种) undefined，null，object，string，number

  • 构造器(9种）object，boolean，string，number，function，array，date，regexp，error

  • 对象(4种) math，json，全局对象，arguments，

  • 以上为原生对象,也是本次讲解的

  • window，document，…(宿主对象)

  • activeXobject，debug，xml(浏览器对象)


一:标准类型



  • 原始类型(5种):undefined，null，string，number

  • 引用类型(object)：通过new+构造器函数创造出来的也是引用类型 如


new String(“xx”);

new Number(123);


二:原始类型与引用类型


var a;//undefined 未赋值,但是a保存在了栈内存中,值为undefined

var b=null// b保存在了栈内存中,值为null

var f={a:1}// f在栈内存中只保存了一个引用地址,这个地址指向的是堆内存所在的区域


三:原始类型与引用类型复制区别


var num1=123;

var num2=num1;

原始类型复制前栈内存为


栈内存

num1 | 123;


复制后栈内存为

num1 | 123;

num2 | 123;


值都是一样的.


var obj1={a:1};

var obj2=obj1;

复制前


栈内存 堆内存1

obj1 | 地址指向堆内存 如堆内存1 a | 1


复制后

obj1 | 地址指向堆内存 如堆内存1

obj2 | 地址指向堆内存 如堆内存1


两者指向的堆内存都是一样的，并不会重新开一个堆内存

所以对obj2进行修改同样也会影响到obj1


四:undefined


-值:undefined


-出现场景


1：已声明为赋值的变量

var a;//undefined


2:获取对象不存在的属性

var o={a:1};

o.y//undefined


3：无返回值的函数执行结果,（匿名函数也无返回值,因为是立即执行)

function ab(){alert(a)}

var ret=ab()

console.log(ret)//undefined


4:函数参数未传入


function m(i,j){

console.log(i);

console.log(j);//undefined

}


m(1),//并未传入两个参数


-类型转换


undefined-boolean:false,


-number:Number(undefined)//Nan


-string:String(“undefined”)//“undefined”


五：null


-值:null


-出现场景:对象不存在的情况下 如 document.getElementById(“xxx”);


-类型转换


null-boolean:false


-number:0;


-string:”null”


六:boolean


-值:true,false


-出现场景:条件语句导致系统执行的隐式类型转化 如: if(){}


-类型转换


true-number:1


-string:”true”;


false-number:0;


-string:false;


七:string


-值:有单引号或者双引号括起来的字符序列


-类型转换


“空”-boolean:false;


-number:0;


“非空”-boolean:true;


-number:1;


八:number


-值: 整形,八进制,十六进制，浮点


-类型转换


0-boolean:false;


-string:”0”;


1-boolean:true;


-string:”1”;


infinity-boolean:true;


-string:”infinity”;


NaN-boolean:false;


-string:”NaN”;


九:object


-object是一组属性集合


{}-boolean:true


-number:NaN;

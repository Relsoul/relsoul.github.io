js程序设计之构造器对象跟普通对象的区别、JS对象原型链的简要说明

构造器对象跟普通对象的区别


以下代码创建了一个Point构造器


function Point(x, y){


this.x = x;


this.y = y;


}


Point.prototype.move = function(x, y){


this.x += x;


this.y += y;


}


在调试中展开Point结构如下图


构造器对象相对于普通对象（回顾一下“对象原型链的说明”中的p对象）有如下几个特点：


构造器对象原型链倒数第二个proto是一个Function.prototype对象引用,因此可以调用Function.prototype的属性和方法。


构造器对象本身有一个prototype属性，用该构造器实例化对象时该prototype会被实例对象的proto所引用。


构造器对象本身是一个function对象，因此会有name,length等自身属性。


 


 


JS对象原型链的简要说明


以下代码创建了一个Point构造器并实例化了一个p对象


function Point(x, y){


this.x = x;


this.y = y;


}


Point.prototype.move = function(x, y){


this.x += x;


this.y += y;


}


var p = new Point(1,1);


p.move(1,2);


在调试中展开p对象结构如下图


“proto“就是我们通常说的原型链属性，他有如下几个特点：


“proto“是对象的一个内部隐藏属性。


“proto“是实例化该对象的构造器的prototype对象的一个引用，因此可以访问prototype的所有属性和方法。


除了Object对象，每个对象都有一个”proto“属性，”proto“逐级增长形成一个链就是我们所说的原型链，原型链顶端是一个Object对象。


当开发者调用对象属性或方法时(比如”p.move(1,2)”)，引擎首先会查找p对象的自身属性，如果自身属性中没有”move”方法，则会继续沿原型链逐级向上查找，直到找到该方法并调用。


“proto“跟浏览器引擎实现相关，不同的引擎中名字和实现不尽相同(chrome、firefox中名称是”proto“，并且可以被访问到，IE中无法访问)。基于代码兼容性、可读性等方面的考虑，不建议开发者显式访问”proto“属性或通过”proto“更改原型链上的属性和方法，可以通过更改构造器prototype对象来更改对象的proto属性。

PS切图之切图

切图


一:修饰性的(一般为CSS属性的background)



  • 图标,Logo

  • 有特殊效果的按钮文字

  • 非纯色背景


二:内容性(一般为HTML的img标签)



  • 广告

  • 配图


三:切出来图片存为那种类型？



  • 修饰性

    - png24（IE6不支持半透明)


    -png8


  • 内容性

    -一般存为jpg



四:切图(隐藏文字只留背景)



  • 若文字为独立图层,则隐藏文字图层大概步骤为 :1.去掉眼睛图标,2:保存

  • 若文字与背景合并

    -背景可以平铺:1.利用矩形选框工具,2:利用自由变换（CTRL+T)覆盖掉文字即可


    -背景不可以平铺:1.利用移动工具+Alt，2.每一次移动一点点,直到覆盖掉整个文字



五:切图·png24



  • 移动工具选择所需图层(按住Ctrl多选)

  • 右键合并图层(Ctrl+E)

  • 再右键复制图层到新文件(新建:Ctrl+N)或者直接拖至已有文件


六:带背景切



  • 合并(可见)图层(Shift+Ctrl+E)

  • 矩形选框工具选择内容

  • 魔棒工具去除多余部分

    -从选区中减去，按住ALT



七:可平铺背景的切图-CSS:background



  • 利用矩形选框工具选区一块区域

  • 复制粘贴到新文件

  • 平铺内容充满文件的宽(X轴)或高(Y轴)


八：切片工具



  • 适用于可以一刀切的活动页

  • 拉参考线

  • 选择切片工具

  • 点击”基于参考性的切片”按钮

  • 保存(全选切片,统一设置存储格式)

    -保存为 WEB所用格式 JPEG(色彩丰富)

折线图的一些说明

tooltip

提示框，鼠标悬浮交互时的信息提示，比如

{bollean}show true|false 显示|隐藏
{number}showDelay 20 显示延迟

其他的信息请看官网范例

legend

图例，每个图表最多仅有一个图例 官网范例

toolbox

工具箱，每个图表最多仅有一个工具箱 官网范例

calculable

拖拽计算

{boolean} false，是否启用拖拽重计算特性，默认关闭

xAis/yAxis

点击标题参考文档

series

驱动图表生成的数据内容数组，数组中每一项为一个系列的选项及数据，其中个别选项仅在部分图表类型中有效，请注意适用类型 点击标题直接去官网看文档吧

        }
    }

    -无访问控制

        action:"add",
        compute:function compute(a,b){
            switch(action){
                case "add":return math.add(a,b)
                case "sub":return math.add(a,b)
            }
        }
    }
    -同样没有依赖声明

                }

产品前端构架之git命令 

一:git help
                -git –help

二:git congfig
    -用户配置
        -git config --global user.name "xxx"
        -git config --global user.email test@xxx.com
    -配置级别
        - --local [默认，高优先级]:只影响本地仓库 .git/config
        - --global [中优先级]:影响到所有当前用户的git仓库 ~/.gitconfig
        - --system [低优先级]:影响到全系统的git仓库 /etc/gitconfig

三:git init
    -git status
        -查看当前仓库信息
    -git init
        -初始化一个git仓库

四:git status
    -内容状态
        -工作目录
        -暂存区
        -提交区
    -了解当前内容状态

五:git add
    -添加文件内容到暂存区（同时文件被跟踪）
    -批量添加
        -git add .  //添加当前目录所有文件

六:忽略文件
    -.gitgnore
        -在添加时忽略匹配的文件
        -仅作用于未跟踪的文件
    -常见的gitgnore
        #logs //注释
        logs
        *.log

七:从暂存区删除git-rm
    -git rm --cached:仅从暂存区删除
    -git rm :从暂存区与工作目录删除
    -git rm $(git ls-files --deleted):删除所有被跟踪，但是在工作目录被删除的文件

八:工作目录与暂存区
    -物品:文件
    -货架:工作目录
    -购物车:暂存区
    -购买:提交内容

    -每类物品只能放置一次的购物车

CSS布局之两列布局 

<!DOCTYPE html>
<html>
    <head>
        <meta charset=”utf-8”>
        <title></title>
        <style type=”text/css”>
            /一列定宽一列自适应之float+margin
            {padding: 0;margin: 0;}
                .left{
                    float: left;
                    width: 100px;
                    background: red;
                }
                .right{
                    margin-left: 100px; /往右缩进100px; 间距通过margin-left设置
                    background: blue;
                }/

            /*二:一列定宽一列自适应之float+margin+fix
            .left{
                float: left;
                width: 100px;
                position: relative;  提高层级
            }
            .right{
                margin-left: 120px;
            }
            .right-fix{
                float:right;
                width: 100%
                margin-left:-100px;
            }*/


        /*三:一列定宽一列自适应之float+overflow
        .left{
            float:left;
            width: 100px;
            margin-right:20px; /*间距设置在left上
        }
        .right{
            overflow: hidden; /*触发bfc模式,里面内容跟外界隔离,实现所需布局
        }*/

        /*四:一列定宽一列自适应之table
        .parent{
            display: table; 宽度默认为auto
            width: 100%; 设置宽度为100%
            table-layout: fixed; 加速table渲染，实现了布局优先
        }
        .left,.right{
            display:table-cell; left和right变为表格
        }
        .left{
            width:100px;
            padding-right: 20px;设置间距
        }*/


    /*五:一列定宽一列自适应之flex
            .parent{
                display: flex;
            }
            .left{
                width: 100px;
                margin-right:20px ;
            }
            .right{
                flex:1;
            }*/


    &lt;/style&gt;
    &lt;script type="text/javascript"&gt;
        一:一列定宽一列自适应之float+margin
            -优点:容易理解
            -IE6文字会往右缩进3px 通过在left加margin-right可以解决
            -在right子元素上清除浮动.会在right元素上空出left高度的一个空白

        二:一列定宽一列自适应之float+margin+fix
            -不会存在3px问题
            -不会存在right子元素上清除浮动也不会影响
            -right-fix会盖在left上面，所以需要提高left层级

        三:一列定宽一列自适应之float+overflow
            -IE6不支持

        四:一列定宽一列自适应之table
            -代码比较多- -

        五:一列定宽一列自适应之flex
            -不兼容低版本
            -性能会有一个问题(flex布局会涉及到计算)
            -适用于小范围布局


    &lt;/script&gt;

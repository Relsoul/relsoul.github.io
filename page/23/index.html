折线图的一些说明

tooltip

提示框，鼠标悬浮交互时的信息提示，比如

{bollean}show true|false 显示|隐藏
{number}showDelay 20 显示延迟

其他的信息请看官网范例

legend

图例，每个图表最多仅有一个图例 官网范例

toolbox

工具箱，每个图表最多仅有一个工具箱 官网范例

calculable

拖拽计算

{boolean} false，是否启用拖拽重计算特性，默认关闭

xAis/yAxis

点击标题参考文档

series

驱动图表生成的数据内容数组，数组中每一项为一个系列的选项及数据，其中个别选项仅在部分图表类型中有效，请注意适用类型 点击标题直接去官网看文档吧

                -依赖管理 加载/分析/注入/初始化
                -决定模块写法

二:Commonjs/module 非浏览器环境的js
    -math.js
        function add(a,b){
            return a+b;
        };
        function sub(a,b){
            return a-b
        }
        exports.add=add
        exports.sub=sub
    -caculator.js
        var math=require("./math");
        function Caculator(container){
            this.add=container.querySelector("xx")
            //do sth
        }
        exports.Caculator=Caculator
        优点
        -依赖管理程度可靠
        -社区活跃，规范接受度高
        -运行时指出，模块定义非常简单
        -文件级的模块作用域隔离
        -可以处理循环依赖
        缺点
        -不是标准组织的规范
        -同步的require，没有考虑浏览器环境

    三:AMD
        -math.js
            define([],function(){
                function add(a,b){
                    return a+b
                }
                function sub(a,b){
                    return a-b
                }
                //接口暴露
                return{
                    add:add,
                    sub:sub
                }
            })

        -caculator.js
        //依赖声明
            define(["./math"],function(math){
                //do sth
                return{
                    Caculator:Caculator
                }
            })
        优点
        -依赖管理成熟可靠
        -社区活跃，规范接受度高
        -转为异步IO环境打造，适合浏览器环境
        -支持类似Commonjs的书写方法
        -通过插件API可支持加载非JS资源
        -成熟的打包构建工具
        缺点
        -模块定义繁琐，需要额外嵌套
        -只是库级别的支持，需要引入额外库
        -无法处理循环依赖
        -无法实现条件加载

    四:systemjs
        -支持加载AMD
        -支持加载Commonjs
        -支持加载ES6
        -支持加载任何资源

JS之探测器实例 

!function(global){
                function DetectorBase(configs){
                    //如果this不是DetectorBase本身的话也就是 this为DetectorBase prototype
                    //必须使用new DetectorBase 阻止外部直接调用
                    if(!this instanceof DetectorBase){
                        throw new Error(“Do not invoke without new|无法调用新detectorBase”)
                    }
                    this.configs=configs;
                    this.analyze();//创建的时候是一个实例 实例.analyze() 解析一下数据
                }
                DetectorBase.prototype.detect=function(){
                    throw new Error(“Not implemented|未执行”)
                }
                DetectorBase.prototype.analyze=function(){
                    console.log(“analyzing..”);
                    this.data=”###date###”
                }
                function LinkDetector(links){
                    if(!this instanceof LinkDetector){
                        throw new Error(“Do not invoke without new|无法调用新detectorBase”)
                    }
                    this.links=links;
                    DetectorBase.apply(this,arguments)
                }
                function ContainerDetector(containers){
                    if(!this instanceof ContainerDetector){
                        throw new Error(“Do not invoke without new|无法调用新detectorBase”)
                    }
                    this.containers=containers
                    DetectorBase.apply(this,arguments);
                }
                //inherit first
                //实际上就是Object.create()
                //先实现原型链继承 再实现link,containerDetector属性
                inherit(LinkDetector,DetectorBase);
                inherit(ContainerDetector,DetectorBase)

    //expand later 扩展
    LinkDetector.prototype.detect=function(){
        console.log("Loading data:"+this.data)
        console.log("Link detection started.")
        console.log("Scaning links:"+this.links)
    }
    ContainerDetector.prototype.detect=function(){
        console.log("Loading data:"+this.data)
        console.log("Container detection started")
        console.log("Scanning containers:"+this.containers)
    }

    //prevent from being altered 防止被修改
    Object.freeze(DetectorBase)
    Object.freeze(DetectorBase.prototype)
    Object.freeze(LinkDetector)
    Object.freeze(LinkDetector.prototype)
    Object.freeze(ContainerDetector)
    Object.freeze(ContainerDetector.prototype)

    //export to global object 导出到全局对象
    //暴露到外面 同时不可改写
    Object.defineProperties(global,{
        LinkDetector:{value:LinkDetector},
        ContainerDetector:{value:ContainerDetector},
        DetectorBase:{value:DetectorBase}
    })
    function inherit(subClass,superClass){
        subClass.prototype=Object.create(superClass.prototype)
        subClass.prototype.constructor=subClass
    }
}(this)

CSS布局之两列布局 

<!DOCTYPE html>
<html>
    <head>
        <meta charset=”utf-8”>
        <title></title>
        <style type=”text/css”>
            /一列定宽一列自适应之float+margin
            {padding: 0;margin: 0;}
                .left{
                    float: left;
                    width: 100px;
                    background: red;
                }
                .right{
                    margin-left: 100px; /往右缩进100px; 间距通过margin-left设置
                    background: blue;
                }/

            /*二:一列定宽一列自适应之float+margin+fix
            .left{
                float: left;
                width: 100px;
                position: relative;  提高层级
            }
            .right{
                margin-left: 120px;
            }
            .right-fix{
                float:right;
                width: 100%
                margin-left:-100px;
            }*/


        /*三:一列定宽一列自适应之float+overflow
        .left{
            float:left;
            width: 100px;
            margin-right:20px; /*间距设置在left上
        }
        .right{
            overflow: hidden; /*触发bfc模式,里面内容跟外界隔离,实现所需布局
        }*/

        /*四:一列定宽一列自适应之table
        .parent{
            display: table; 宽度默认为auto
            width: 100%; 设置宽度为100%
            table-layout: fixed; 加速table渲染，实现了布局优先
        }
        .left,.right{
            display:table-cell; left和right变为表格
        }
        .left{
            width:100px;
            padding-right: 20px;设置间距
        }*/


    /*五:一列定宽一列自适应之flex
            .parent{
                display: flex;
            }
            .left{
                width: 100px;
                margin-right:20px ;
            }
            .right{
                flex:1;
            }*/


    &lt;/style&gt;
    &lt;script type="text/javascript"&gt;
        一:一列定宽一列自适应之float+margin
            -优点:容易理解
            -IE6文字会往右缩进3px 通过在left加margin-right可以解决
            -在right子元素上清除浮动.会在right元素上空出left高度的一个空白

        二:一列定宽一列自适应之float+margin+fix
            -不会存在3px问题
            -不会存在right子元素上清除浮动也不会影响
            -right-fix会盖在left上面，所以需要提高left层级

        三:一列定宽一列自适应之float+overflow
            -IE6不支持

        四:一列定宽一列自适应之table
            -代码比较多- -

        五:一列定宽一列自适应之flex
            -不兼容低版本
            -性能会有一个问题(flex布局会涉及到计算)
            -适用于小范围布局

来一个最简单的 Hello World 吧！


 


我们一般接触到的第一个入门教程都是类似与这样的，代码简单。我们有个ng-app作为Angular的启动口，然后ng-controller=”helloCtrl”控制p标签。View层，则用双花括号出入数据greeting的message值。然后在下面的script定义了一个helloCtrl的函数，并为$scope.greeting赋了值。这一切看起来都是那么自然，想象中，浏览器就该出现，”Hello, World!”。可惜的是，浏览器给了我们这样的回复。


没错，这就是开头所说的那个。于是我弄不懂，为什么老师可以跑通，我却跑不通，大家都是Angular-1.3.0。国内不给力，很难找到答案，最后去StackOverflow找到了。因为从Angular-1.3.0 beta15开始，Angular不再支持全局的Controller！那么问题来了，如何在Angular里定义模块呢？


当然这篇blog是为了让大家学习Angular的时候别遇到闭门羹，我大致说一下。AngularJs里，有一个angular.module，模块机制就是靠这个来加载的。我们来看看如何可以重新出现梦寐以求的“Hello World”。


我们看看，改了什么，ng-app=”myApp”首先给入口赋了名字，然后再下面进行模块的初始化。在angular.module(String, moduleArray)函数里，第一个String参数是该模块的名字，第二个参数是所依赖的模块（这里暂时用不着），返回了myApp这个模块，然后再通过模块的controller方法定义helloCtrl控制器即可。


 


当然我们要解释依赖模块的话，我们可以看一下稍作修改的第三版Demo


 


仅仅是多定义一个helloController模块，这样myApp，就可以仅仅作为启动口，而不会混杂着控制器。我们可以定义多个js文件，清晰明确的通过模块梳理整个项目的结构。

        居中布局 父元素 子元素宽/高度不固定

        一：水平居中之inline-block+text-align
            -兼容性非常好
            -inline+zoom:1  //兼容IE8以下
            -child会继承text-align:center，需要对子元素进行reset

            -只需要设置child ,IE6 7可以child可以采用table的方式如th tr来实现

        三:水平居中之absolute+transform
            -脱离文档流 不会对其他元素产生影响
            -不兼容低版本IE

            }

            -兼容低版本HTML结构换成table结构

        六:垂直居中之absolute+transform
            -子元素不会干扰其他元素

        七:垂直居中之flex+align-items
            -不兼容低版本IE

        八:居中(水平垂直居中)之inline-block+text-align+table-cell+vertical-align

CSS布局之规范 

一:规范
                -文件规范
                -注释规范
                -命名规范
                -书写规范
                -其他规范

二:文件规范
    -文件分类
        -通用类
            -reset.css jquery-ul.css
        -业务类
            -alibum toplist
    -文件引入
        -行内样式
            -不推荐
        -外联引入
            -link
        -内联引入
            -style
        -避免在CSS中使用import
    -文件本身
        -文件名
            -由小写字母，数字，中划线组成
        -编码
            -utf-8

三:注释规范
    -块状注释
        -统一缩进
        -在被注释对象之上
    -单行注释
        -文字两端空格
        -在被注释对象之上
    -行内注释
        -文字两端空格
        -在分号之后

四:命名规范
    -分类命名
        -g-header
        -h-header
    -命名格式
        -大小写
        -长度
            -衡量长度和可读性
    -语义化命名
        -以内容语义命名

五:书写规范
    -单行与多行
        -.logo{//do sth}
        -.logo{
            padding://
            color://
        }
        -二选一 遵守之
    -空格与分号
        -空格
            -缩进 2 or 4
            -规则内空格
                -.logo{width: 200px;height: 50px;}
        -分号
            -保留最后一个属性值的分号
    -属性顺序
        -根据属性的重要性按顺序书写
            -显示属性
            -自身属性
            -文本属性和其他修饰
    -Hack方式
        -统一各个浏览器的Hack方式
            -IE6:_pro:value
            -IE7:*pro:value
        -不要滥用hack
    -值格式
        -统一属性值
            -color
                -white or #fff or rgb(255,255,255)
            -url()
                -icon.png or "icon.png" or 'icon.png'
六:其他规范
    -HTML规范
        -文档声明
        -闭合
            -闭合标签
            -自闭合标签
        -属性
            -双引号与单引号
            -&lt;input readonly&gt; or &lt;input readonly="readonly"&gt;
        -层级
            -用缩进体现层级，提高可读性
            -标签正确嵌套，但嵌套不宜太深
        -注释
            -&lt;!-- LOGO --&gt;
            -&lt;!-- /LOGO--&gt;
        -大小写
            -标签，属性均小写

    -图片规范
        -文件名称
            -语义
            -长度
        -保留源文件
        -图片合并
            -尽量使用sprite技术
            -sprite图片可按模块，业务，页面来划分</pre>

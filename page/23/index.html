产品前端构架之git命令 

一:git help
                -git –help

二:git congfig
    -用户配置
        -git config --global user.name "xxx"
        -git config --global user.email test@xxx.com
    -配置级别
        - --local [默认，高优先级]:只影响本地仓库 .git/config
        - --global [中优先级]:影响到所有当前用户的git仓库 ~/.gitconfig
        - --system [低优先级]:影响到全系统的git仓库 /etc/gitconfig

三:git init
    -git status
        -查看当前仓库信息
    -git init
        -初始化一个git仓库

四:git status
    -内容状态
        -工作目录
        -暂存区
        -提交区
    -了解当前内容状态

五:git add
    -添加文件内容到暂存区（同时文件被跟踪）
    -批量添加
        -git add .  //添加当前目录所有文件

六:忽略文件
    -.gitgnore
        -在添加时忽略匹配的文件
        -仅作用于未跟踪的文件
    -常见的gitgnore
        #logs //注释
        logs
        *.log

七:从暂存区删除git-rm
    -git rm --cached:仅从暂存区删除
    -git rm :从暂存区与工作目录删除
    -git rm $(git ls-files --deleted):删除所有被跟踪，但是在工作目录被删除的文件

八:工作目录与暂存区
    -物品:文件
    -货架:工作目录
    -购物车:暂存区
    -购买:提交内容

    -每类物品只能放置一次的购物车
    -货架和购物车可以出现同种物品
    -货架上的物品可以替换掉购物车物品
    -可以删除物品
    -提交购物车完成购买，生成购买记录

九:git commit
    -根据暂存区内容创建一个提交记录
    -git commit -m ""
    -直接提交
        -git commit -a

十:git log
    -git log --oneline
        -简单log

十一:git中的alias(别名）命令
        -git config alias.shortname &lt;fullcommad&gt;
        -git config --global alias.lg "log --oneline"
        -git lg//配置了别名

十二:git diff
    -显示不同的版本差异
    -git diff:
        -工作目录与暂存区的差异
    -git diff --cached
        -暂存区与某次提交差异,默认为HEAD
    -git diff &lt;reference&gt;
        -工作目录与某次提交差异

十三:撤销本地修改
    -git checkout --&lt;file&gt;
        -将文件内容从暂存区复制到工作目录

十四:撤销暂存区内容
    -git reset HEAD &lt;file&gt;
        -将文件内容从上次提交复制到暂存区

十五:撤销全部改动
    -git checkout HEAD --&lt;file&gt;
    -将内容从上次提交复制到工作目录</pre>

JS之探测器实例 

!function(global){
                function DetectorBase(configs){
                    //如果this不是DetectorBase本身的话也就是 this为DetectorBase prototype
                    //必须使用new DetectorBase 阻止外部直接调用
                    if(!this instanceof DetectorBase){
                        throw new Error(“Do not invoke without new|无法调用新detectorBase”)
                    }
                    this.configs=configs;
                    this.analyze();//创建的时候是一个实例 实例.analyze() 解析一下数据
                }
                DetectorBase.prototype.detect=function(){
                    throw new Error(“Not implemented|未执行”)
                }
                DetectorBase.prototype.analyze=function(){
                    console.log(“analyzing..”);
                    this.data=”###date###”
                }
                function LinkDetector(links){
                    if(!this instanceof LinkDetector){
                        throw new Error(“Do not invoke without new|无法调用新detectorBase”)
                    }
                    this.links=links;
                    DetectorBase.apply(this,arguments)
                }
                function ContainerDetector(containers){
                    if(!this instanceof ContainerDetector){
                        throw new Error(“Do not invoke without new|无法调用新detectorBase”)
                    }
                    this.containers=containers
                    DetectorBase.apply(this,arguments);
                }
                //inherit first
                //实际上就是Object.create()
                //先实现原型链继承 再实现link,containerDetector属性
                inherit(LinkDetector,DetectorBase);
                inherit(ContainerDetector,DetectorBase)

    //expand later 扩展
    LinkDetector.prototype.detect=function(){
        console.log("Loading data:"+this.data)
        console.log("Link detection started.")
        console.log("Scaning links:"+this.links)
    }
    ContainerDetector.prototype.detect=function(){
        console.log("Loading data:"+this.data)
        console.log("Container detection started")
        console.log("Scanning containers:"+this.containers)
    }

    //prevent from being altered 防止被修改
    Object.freeze(DetectorBase)
    Object.freeze(DetectorBase.prototype)
    Object.freeze(LinkDetector)
    Object.freeze(LinkDetector.prototype)
    Object.freeze(ContainerDetector)
    Object.freeze(ContainerDetector.prototype)

    //export to global object 导出到全局对象
    //暴露到外面 同时不可改写
    Object.defineProperties(global,{
        LinkDetector:{value:LinkDetector},
        ContainerDetector:{value:ContainerDetector},
        DetectorBase:{value:DetectorBase}
    })
    function inherit(subClass,superClass){
        subClass.prototype=Object.create(superClass.prototype)
        subClass.prototype.constructor=subClass
    }
}(this)

DOM编程艺术之数据通信-HTTP协议 

一:HTTP
            -客户端向服务器发送包含命令和URL的HTTP请求报文
            -服务器返回一个包含事务结果的HTTP响应报文

二:请求报文格式
    -头行 
        -GET music.163.com HTTP/1.1
          方式  主机地址        版本
    -头部
        -许多键值对构成
            -Accept:浏览器端接受的类型
            -Accept-Encoding:浏览器端编码类型
            -Accept-Language:浏览器端可以接受的语言类型
            -Cache-Control:浏览器是否可以缓存
            -Connection:是否持续链接
            -Cookie:浏览器向服务器端发送的cookie
            -Host:主机地址
            -Pragma:
            -User-Agent:浏览器版本

三:响应报文格式
    -头行
        -HTTP/1.1  200    OK
         HTTP版本  状态码    HTTP状态码描述
    -头部
        -Expires:缓存有效时间
        -Server:服务器端HTTP服务器
    -主体
        -完整的XML页面

四:常用HTTP方法
    -GET
        -从服务器获取一份稳定 不包含主体
    -POST
        -向服务器发送需要处理的数据 包含主体
    -PUT
        -将请求的主体部分储存在服务器上 包含主体
    -DELETE
        -从服务器上删除一份稳定 不包含主体
    -HEAD
        -只从服务器获取文档的首部 不包含主体
    -TRACE
        -对可能经过代理服务器传送到服务器上的报文进行追踪 不包含主体
    -OPTIONS
        -决定可以在服务器上执行哪些方法 不包含主体

五:URL构成
    -http://www.163.com:8080/index.html?=admin&amp;lang=zh-CN#news
    -protocol
        -http
    -host
        -www.163.com:8080
    -pathname
        -index.html
    -hostname
        -www.163.com
    -port
        -8080
    -search
        -?r=admin&amp;lang=zh-CN
    -hash
        -news

六:HTTP版本
    -HTTP/0.9
        -1991年 HTTP原型，有很多设计缺陷
    -HTTP/1.0
        -很快取代HTTP/0.9，成为第一个广泛应用的版本
    -HTTP/1.0+
        -添加持久的keep-alive连接，虚拟主机支持，以及代理连接支持，成为非官方的事实标准
    -HTTP/1.1
        -校正了结构性缺点，明确语义，引入重要的性能优化措施，删除不好特性，是当前使用的版

七:常见HTTP状态码
    -200
        -请求成功。一般用于GET和POST方法 原因短语:OK
    -301
        -资源移动，所请求资源自动到新的URL，浏览器自动跳转到新的URL moved Permanently
    -304
        -未修改。所请求资源未修改，浏览器读取缓存数据 Not Modified
    -400
        -请求语法错误，服务器无法理解 Bad Request
    -404
        -未找到资源，可以设置个性404页面 Not Found
    -500
        -内部服务器错误 Internal Server Error

    [

var myChart=ec.init(document.getElementById("main"));

第五:绘图

myChart.setOption(option);

一个标准折线图图表有这几大元素

第一:标题

//标题
                        title:{
                            //文本
                            text:"未来一周气温变化",
                            //子标题
                            subtext:"纯属虚构"
                        },//!-文本

第二:鼠标移入上去的提示框

//提示框
                        tooltip:{
                            //坐标轴触发显示
                            trigger:"axis"

                        },

第三:图例

//图例 每个图表只能有一个
                            legend:{
                                data:['最高气温','最低气温']
                            },//!-图例

第五:工具箱

//工具箱 每个图表只能有一个
                            toolbox:{
                                //是否显示工具箱 图表右上角的一系列开关
                                show:true,
                                //特效duang
                                feature:{
                                    //辅助线标志
                                    mark:{show:true},
                                    //数据视图 可以显示文本数据 
                                    //是否仅读？NO
                                    dataView:{show:true,readOnly:false},
                                    //可以切换更多的图形特效吗？
                                    //YSE! 可以切换line和bar类型
                                    magicType:{show:true,type:["line",'bar']},
                                    //卧槽！按错了，可以复原吗？
                                    //YES!
                                    restore:{show:"true"},
                                    //是否可以保存为图片？
                                    //YES!图片可以嵌入PPT里哦~
                                    saveAsImage:{show:true}
                                },//！-特效

                            },//!-工具箱

第六:横坐标

xAxis:[
                                {
                                    //X轴为类目型
                                    type:"category",
                                    //是否两端留白？
                                    boundaryGap:false,
                                    //横坐标数据 
                                    //只有类目型的才配拥有data属性！
                                    data:["周一","周二","周三","周四","周五","周六","周日"]

                                }
                            ],//!-横坐标

第七:纵坐标

yAxis:[
                                {
                                    //Y轴类型为数值型
                                    type:"value",
                                    //Y轴的文本标签

                                    axisLabel:{
                                        //间隔名称..实际上就是单位 
                                        //{value}传递过来的就是值

                                    }
                                }
                            ],
                            //!-纵坐标

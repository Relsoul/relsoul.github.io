七:attribute之类型
    -input.getAttribute('onclick')//string
    -input.getAttribute("disabled")//string
    -input.getAttribute("maxlength")//string
    -仅仅是字符串

八:实例
    -button.disabled=true;
    -button.setAttribute('class', 'disabled')

九:dataset
    -data-*属性集
    -元素上保存数据

    var math={

        }
    }

    -无访问控制

    var caculator={
        action:"add",
        compute:function compute(a,b){
            switch(action){
                case "add":return math.add(a,b)
                case "sub":return math.add(a,b)
            }
        }
    }
    -同样没有依赖声明

    -caculator-1.js

                    }
                }
            }

        -var caculator=(function(m){

产品前端构架之模块系统 

一：模块系统
                -依赖管理 加载/分析/注入/初始化
                -决定模块写法

二:Commonjs/module 非浏览器环境的js
    -math.js
        function add(a,b){
            return a+b;
        };
        function sub(a,b){
            return a-b
        }
        exports.add=add
        exports.sub=sub
    -caculator.js
        var math=require("./math");
        function Caculator(container){
            this.add=container.querySelector("xx")
            //do sth
        }
        exports.Caculator=Caculator
        优点
        -依赖管理程度可靠
        -社区活跃，规范接受度高
        -运行时指出，模块定义非常简单
        -文件级的模块作用域隔离
        -可以处理循环依赖
        缺点
        -不是标准组织的规范
        -同步的require，没有考虑浏览器环境

    三:AMD
        -math.js
            define([],function(){
                function add(a,b){
                    return a+b
                }
                function sub(a,b){
                    return a-b
                }
                //接口暴露
                return{
                    add:add,
                    sub:sub
                }
            })

        -caculator.js
        //依赖声明
            define(["./math"],function(math){
                //do sth
                return{
                    Caculator:Caculator
                }
            })
        优点
        -依赖管理成熟可靠
        -社区活跃，规范接受度高
        -转为异步IO环境打造，适合浏览器环境
        -支持类似Commonjs的书写方法
        -通过插件API可支持加载非JS资源
        -成熟的打包构建工具
        缺点
        -模块定义繁琐，需要额外嵌套
        -只是库级别的支持，需要引入额外库
        -无法处理循环依赖
        -无法实现条件加载

    四:systemjs
        -支持加载AMD
        -支持加载Commonjs
        -支持加载ES6
        -支持加载任何资源
    System.paths={
        "app/*":"../es6/*.js"
    }

产品前端构架之页面优化 

一:为什么要优化
                -提升网页相应速度
                -对搜索引擎，屏幕阅读器友好
                -提高可读性，可维护性

二:如何优化
    -减少请求
    -减少文件大小
    -页面性能
    -可读性，可维护性

三:减少请求
    -图片合并
    -CSS文件合并
        -多个CSS文件合并为一个
        -少量CSS样式内联
        -避免使用import的方式引入css文件

四:减少文件大小
    -减少图片大小
        -选择合适的图片格式
            -PNG
            -JPG
        -压缩图片
            -ImageOptim
            -ImageAlpha

        -CSS值缩写
            p{
                margin:10px 10px 20px 30px;
            }
        -省略值为0的单位
        -颜色值最短表示
            color:#ccc
        -CSS选择器合并
        -文件压缩
            -YUI Compressor
            -cssmin

五:页面性能
    -加载顺序
        -Css文件放在head里
        -script放在body底部
    -减少标签数量
    -选择器长度
    -耗性能属性
        -expression
        -filter
        -border-radius
        -box-shadow
        -gradients
    -图片设置宽高
    -所有表现用CSS实现
        elm.className="shrink"
        .shrink li{width:100px}

六:可读性，可维护性
    -规范
    -语义化
    -尽量避免Hack

    -注释</pre>

CSS布局之规范 

一:规范
                -文件规范
                -注释规范
                -命名规范
                -书写规范
                -其他规范

二:文件规范
    -文件分类
        -通用类
            -reset.css jquery-ul.css
        -业务类
            -alibum toplist
    -文件引入
        -行内样式
            -不推荐
        -外联引入
            -link
        -内联引入
            -style
        -避免在CSS中使用import
    -文件本身
        -文件名
            -由小写字母，数字，中划线组成
        -编码
            -utf-8

三:注释规范
    -块状注释
        -统一缩进
        -在被注释对象之上
    -单行注释
        -文字两端空格
        -在被注释对象之上
    -行内注释
        -文字两端空格
        -在分号之后

四:命名规范
    -分类命名
        -g-header
        -h-header
    -命名格式
        -大小写
        -长度
            -衡量长度和可读性
    -语义化命名
        -以内容语义命名

五:书写规范
    -单行与多行
        -.logo{//do sth}
        -.logo{
            padding://
            color://
        }
        -二选一 遵守之
    -空格与分号
        -空格
            -缩进 2 or 4
            -规则内空格
                -.logo{width: 200px;height: 50px;}
        -分号
            -保留最后一个属性值的分号
    -属性顺序
        -根据属性的重要性按顺序书写
            -显示属性
            -自身属性
            -文本属性和其他修饰
    -Hack方式
        -统一各个浏览器的Hack方式
            -IE6:_pro:value
            -IE7:*pro:value
        -不要滥用hack
    -值格式
        -统一属性值
            -color
                -white or #fff or rgb(255,255,255)
            -url()
                -icon.png or "icon.png" or 'icon.png'
六:其他规范
    -HTML规范
        -文档声明
        -闭合
            -闭合标签
            -自闭合标签
        -属性
            -双引号与单引号
            -&lt;input readonly&gt; or &lt;input readonly="readonly"&gt;
        -层级
            -用缩进体现层级，提高可读性
            -标签正确嵌套，但嵌套不宜太深
        -注释
            -&lt;!-- LOGO --&gt;
            -&lt;!-- /LOGO--&gt;
        -大小写
            -标签，属性均小写

    -图片规范
        -文件名称
            -语义
            -长度
        -保留源文件
        -图片合并
            -尽量使用sprite技术
            -sprite图片可按模块，业务，页面来划分</pre>

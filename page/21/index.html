-默认宽度为内容宽度

媒体查询:支持不同的视口

媒体查询:支持不同的视口

支持程度

IE9+,FF3.6+,Chrome4+,Safari4+,Oprea9.5+,Android2.1+,IOS safari3.2+


媒体查询能检测的特效(摘选)



  • width:视口宽度

  • height:视口高度

  • device-width:设备屏幕宽度

  • device-height:设备屏幕高度

  • orientation:设备处于横向还是纵向


一个媒体查询


//import 会增加HTTP请求，影响加载速度
//屏幕宽度在200px-360px之间
import url(“phone.css”) screen and (min-width:200px) and (max-width:360px);

低版本兼容库


Respond.js为IE8以及更低版本增加媒体查询功能


从固定设计着手


从固定的960px设计开始设计window的效果,同时保持设计中图片的精简


阻止浏览器自动调整页面大小


//放大两倍的设备屏幕宽度效果
//initial-scale=2.0 放大两倍
//width=device-width 设备宽度
<meta name=”viewport” content=”initial-scale=2.0,width=device-width”>

控制可缩放范围与禁止缩放


//maxinum-scale=3 用户最大调整缩放为3倍
//minimun-scale=0.5 用户最小调整缩放为0.5倍
//user-scalable=no 禁止用户缩放
<meta name=”viewport” content=”maxinum-scale=3,minimun-scale=0.5,width=device-width，user-scalable=no”>

针对不同的视口宽度修正设计


//最大宽度为768px的设备进行调整
@media  screen and (max-width: 768px) {
                #nav{
                //do sth
                }
            }

响应式设计中内容始终优先


多平台多视口的情况下保留尽可能多的内容，而不是使用display:none或者类似的方法来隐藏内容


媒体查询只是必要条件之一


媒体查询只能根据设备特性应用对应的样式.一但超出应用样式的范围，媒体查询将无法生效如上面那段代码 如果屏幕宽度大于768px那么将无法生效


我们需要流动布局


如果针对已知的特定访问设备，可以单独使用媒体查询来制作理想的设计效果，我们已经见过配适768px有多简单，但是这种策略有严重的缺陷，换句话说，它不能适应未来的变换。 目前的效果更像是一个自适应，而不是我们想要的响应式。我们设计应该在突变之前保持灵动.

DOM编程艺术之数据通信-HTTP协议 

一:HTTP
            -客户端向服务器发送包含命令和URL的HTTP请求报文
            -服务器返回一个包含事务结果的HTTP响应报文

二:请求报文格式
    -头行 
        -GET music.163.com HTTP/1.1
          方式  主机地址        版本
    -头部
        -许多键值对构成
            -Accept:浏览器端接受的类型
            -Accept-Encoding:浏览器端编码类型
            -Accept-Language:浏览器端可以接受的语言类型
            -Cache-Control:浏览器是否可以缓存
            -Connection:是否持续链接
            -Cookie:浏览器向服务器端发送的cookie
            -Host:主机地址
            -Pragma:
            -User-Agent:浏览器版本

三:响应报文格式
    -头行
        -HTTP/1.1  200    OK
         HTTP版本  状态码    HTTP状态码描述
    -头部
        -Expires:缓存有效时间
        -Server:服务器端HTTP服务器
    -主体
        -完整的XML页面

四:常用HTTP方法
    -GET
        -从服务器获取一份稳定 不包含主体
    -POST
        -向服务器发送需要处理的数据 包含主体
    -PUT
        -将请求的主体部分储存在服务器上 包含主体
    -DELETE
        -从服务器上删除一份稳定 不包含主体
    -HEAD
        -只从服务器获取文档的首部 不包含主体
    -TRACE
        -对可能经过代理服务器传送到服务器上的报文进行追踪 不包含主体
    -OPTIONS
        -决定可以在服务器上执行哪些方法 不包含主体

五:URL构成
    -http://www.163.com:8080/index.html?=admin&amp;lang=zh-CN#news
    -protocol
        -http
    -host
        -www.163.com:8080
    -pathname
        -index.html
    -hostname
        -www.163.com
    -port
        -8080
    -search
        -?r=admin&amp;lang=zh-CN
    -hash
        -news

六:HTTP版本
    -HTTP/0.9
        -1991年 HTTP原型，有很多设计缺陷
    -HTTP/1.0
        -很快取代HTTP/0.9，成为第一个广泛应用的版本
    -HTTP/1.0+
        -添加持久的keep-alive连接，虚拟主机支持，以及代理连接支持，成为非官方的事实标准
    -HTTP/1.1
        -校正了结构性缺点，明确语义，引入重要的性能优化措施，删除不好特性，是当前使用的版

七:常见HTTP状态码
    -200
        -请求成功。一般用于GET和POST方法 原因短语:OK
    -301
        -资源移动，所请求资源自动到新的URL，浏览器自动跳转到新的URL moved Permanently
    -304
        -未修改。所请求资源未修改，浏览器读取缓存数据 Not Modified
    -400
        -请求语法错误，服务器无法理解 Bad Request
    -404
        -未找到资源，可以设置个性404页面 Not Found
    -500
        -内部服务器错误 Internal Server Error

先聊聊JS中的正则


JS中的正则有两种风格





  1. var jsfg=new RegExp(“a”,“gi”) //js风格

  2. var perlfg=/a/gi //perlfg风格





JS风格是通过一个正则构造器来构造的 new RegExp(“正则”,”是否区分大小写，全局搜索等等~”)


perlfg风格通过 /正则/是否区分大小写，全局搜索等等~ 这样的形式

i:不区分大小写 不加就只匹配小写a

match这个函数的功能是以数组的形式返回匹配到的元素.


/apc|bpc|cpc/g 中”|”是什么意思？

这里的/[abc]pc/g 啥意思呢？


“g”我们知道是全局匹配

  1. var re8=/\D/ //注意我这里是大写的\D 不是小写的\d




大写小写肯定是有区别的 他们之间区别也很简单

okay 先别急着看 我们先了解正则火星文中的量词.说白了就是出现次数



量词:个数

{n}正好出现n次

[1-9]\d{8}只出现八次的数字 第一位为1-9数字

{n,m}最少n次最多m次

[1-9]\d{4,10}

{n,}最少n次，最多不限

+{1,}

?{0,1}最少0次最多一次

*{0,}可以有 最多可以有无限 0次也算 匹配不推荐用

  1. /(^\w+)@([a-z0-9]+).([a-z]+$)/gi






这里的()起一个分组的作用 你可以看成数学中的加减乘除所以上例的正则去掉圆括号意思其实是一样的 先分析@前面的正则





  1. (^\w+) //=^\w+





\w是啥意思？匹配任何字类字符，包括下划线。与“[A-Za-z0-9]”等效。 所以邮箱的@前可以有以上的字符并且出现的次数都不限定最少出现一次 但是我为什么要在正则前面加一个^ 这个问题放在后面再说 咋们继续






  1. ([a-z0-9]+). //=[a-z0-9]+.





这里用到了一个.意思为转义字符 我们知道正则中.是匹配任意字符，我们只需要匹配一个单纯的”.”号所以需要用到一个转义字符\ 当然匹配[]也是需要用到转义字符的 [ ]

@gtb123.xxx 这里的gtb123则是匹配到的内容 我们知道邮箱@号后面可以跟数字和字符所以我们要写上可以匹配到数字和字符

  1. ([a-z]+$) //=[a-z]+$





我们知道.后面只能跟英文 比如 .com .cn .net

这里的$是啥意思？ 这里的$代表字符串的结尾 所以这个正则的意思就是 字符串的结尾必须 com net cn 英文结尾的

  1. (^\w+) //=^\w+





这里的^不是代表否,非的意思了 在方括号里面[^0-9]的^代表否，非 那么没在方括号里面的代表的就是字符串开头

  1. /(^\w+)@([a-z0-9]+).([a-z]+$)/gi

  2. 下划线,数字,字母@字母,数字.英文

  3. 比如

  4. test1213_123@AbDd.cn

圆括号不仅仅有分组的意思 还涉及到一个捕获,我们知道match会返回一个数组 举个简单例子





  1. var str19=“2030”;

  2. var re19=/\d+[+-\\/]\d+/

  3. var mat1=str19.match(re19)

  4. console.log(mat1) //[“2030”, index: 0, input: “2030”]




上面是一串匹配数字,符号,数字的正则 看起来比较奇怪 但是还是能够匹配到的. 返回的是一个数组 注意！这时我们加上一个圆括号来分组下





  1. var str19=“2030”;

  2. var re19=/(\d+)([+-\\/])(\d+)/

  3. var mat1=str19.match(re19)

  4. console.log(mat1) //[“2030”, “20”, ““, “30”, index: 0, input: “2030”]




这里返回的数组是[“2030”, “20”, ““, “30”, index: 0, input: “2030”]


0:”2030”


1:”20”


2:”“


3:”30”


可以得出数组[0]为匹配元素 从1开始就得出了分组捕获的内容


上面的正则第一个圆括号分组捕获的是符号前面的数字 第二个圆括号捕获的是符号 第三个圆括号捕获的是符号后面的数字


捕获组还能反向引用，即在表达式中直接使用某个分组的内容。





  1. var str20=“20-20”;

  2. var re20=/(\d+)([+-*\/])(\1)/

  3. var mat2=str20.match(re20)

  4. console.log(mat2) //[“20-20”, “20”, “-“, “20”, index: 0, input: “20-20”]




这里第三个分组 也就是(\1)引用的是第一个分组(\d+)捕获到的内容 同理 如果要引用分组2捕获到的内容那么





  1. var str20=“20-20”;

  2. var re20=/(\d+)([+-*\/])(\2)/

  3. var mat2=str20.match(re20)

  4. console.log(mat2) //[“20-20”, “20”, “-“, “20”, index: 0, input: “20-20”]




改为\2即可 同理 引用3,4,5,… \n即可 别忘记了分组捕获是从1开始的 0为匹配内容


置换前、后两个数的位置





  1. var str21=“178-334”;

  2. var re21=/(\d+)([+-*\/])(\d+)/

  3. var rep3=str21.replace(re21,“$3$2$1”)

  4. console.log(rep3)//334-178




在replace函数中用$1,2,3,n代表分组

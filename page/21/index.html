PS切图之切图

切图


一:修饰性的(一般为CSS属性的background)



  • 图标,Logo

  • 有特殊效果的按钮文字

  • 非纯色背景


二:内容性(一般为HTML的img标签)



  • 广告

  • 配图


三:切出来图片存为那种类型？



  • 修饰性

    - png24（IE6不支持半透明)


    -png8


  • 内容性

    -一般存为jpg



四:切图(隐藏文字只留背景)



  • 若文字为独立图层,则隐藏文字图层大概步骤为 :1.去掉眼睛图标,2:保存

  • 若文字与背景合并

    -背景可以平铺:1.利用矩形选框工具,2:利用自由变换（CTRL+T)覆盖掉文字即可


    -背景不可以平铺:1.利用移动工具+Alt，2.每一次移动一点点,直到覆盖掉整个文字



五:切图·png24



  • 移动工具选择所需图层(按住Ctrl多选)

  • 右键合并图层(Ctrl+E)

  • 再右键复制图层到新文件(新建:Ctrl+N)或者直接拖至已有文件


六:带背景切



  • 合并(可见)图层(Shift+Ctrl+E)

  • 矩形选框工具选择内容

  • 魔棒工具去除多余部分

    -从选区中减去，按住ALT



七:可平铺背景的切图-CSS:background



  • 利用矩形选框工具选区一块区域

  • 复制粘贴到新文件

  • 平铺内容充满文件的宽(X轴)或高(Y轴)


八：切片工具



  • 适用于可以一刀切的活动页

  • 拉参考线

  • 选择切片工具

  • 点击”基于参考性的切片”按钮

  • 保存(全选切片,统一设置存储格式)

    -保存为 WEB所用格式 JPEG(色彩丰富)

页面制作之CSS布局

CSS布局

布局-元素的摆放模式


一: display //设置元素的显示方式

-block

-块级元素

-默认宽度为父元素宽度

-可设置宽高

-换行显示

-默认display:div,p,h1-h6,ul,form

-inline

-默认宽度为内容宽度

-不可设置宽高

-同行显示

-默认inline:span,a,label,cite,em


-inline-block

-默认宽度为内容宽度

-可设置宽高

-同行显示

-整块换行

-默认inline-block:input,textarea,select,button


-none

-设置元素不显示

-visibility：hidden //仍然在页面占据空间


二:块级元素水平居中

-margin:auto;width:978px;


三:居中导航

-通过ul li来设置HTML

-ul{text-align:center;height:30px;line-height:30px;}

-li,a{display:inline-block;width:80px;height:100%;}

——————————————————————————–

四:position 定位

-top,right,bottom,left,z-index -设置位置

-没有设置宽高 那么设置Position会被撑开

-元素边缘边缘距离参造物边缘的距离

-z-index

-Z轴上的排序

-z-index栈

-子级元素的Z-index是参考于父级的Z-index


五:position

-static|relative|absolute|fixed

-positon:relative;//相对定位

-仍在文档流中

-参造物为元素本身

-使用场景:绝对定位的参造物

-positon:absolute;//绝对定位

-默认宽度为内容宽度

-脱离了文档流

-参造物为第一个定位祖先/根元素

-position:fixed //固定定位

-默认宽度为内容宽度

-脱离文档流

-参造物为视窗


六:遮罩

-position:fixed;z-index:999;height:100%;width:100%


七:三行自适应布局

-.body{position:absolute;top:100px;left:0;bottom:100px;right:0;oveflow:auto}


八:float

-left|right|

-默认宽度为内容宽度

-脱离文档流

-向指定方向一直移动

-float的元素在同一文档流中

-float元素半脱离文档流

-对元素，脱离文档流;对内容，在文档流


九:clear

- both|left|right

-应用于后续元素

-应用于后续元素

-clearfix:after{content:”.”;display:block;clear:both;oveflow:hidden;}//在父级上加


十:两列布局

-边栏向右浮动,主栏向左浮动,给父级元素加上清除clearfix


三行自适应布局(position)+两列布局(flot)->两列自适应布局


十一:Flex布局(弹性布局)

-flex container 内容

-flex item 元素

-main axis 主轴

-cross axis 副轴


十二:flex item

-在文档流中的子元素 (position 除外) float也算文档流 ,子元素的子元素也除外


十三:flex-direction 方向

-row(行)|row-reverse(反向行)|column(列)|column-reverse(反向列)


十四:flex-wrap 换行

-nowarp|warp|wrap-reverse


十五:flex-flow 一次性设置方向与换行

-<flex-direction>||<flex-wrap>


十六:order 排列顺序

-order:<number>

-initial:0


十七:flex-basis 弹性基准

-设置flex item的初始宽/高

-flex-basis:main-size|<width>


十八:flex-grow 分配到剩余空间的比例 考虑flex-basis

-flex-grow:<number>

-initial:0


弹性布局元素宽度:flex-basis+flow-grow/sum(flow-grow)remain


十九:flex-shrink 设置或检索弹性盒的收缩比率。

-flex-shrink:<number>

-initial:1

弹性布局元素宽度:flex-basis+flex-shrink/sum(flow-grow)remain


二十:flex

-<flex-grow>||<flex-shrink>||<flex-basis>

伸展因子 收缩因子 基准


二十一:justify-content 对齐剩余空间分配

-flex-start|flex-end|center|space-between|space-around


二十二:align-items

-设置cross-axis方向上对齐方式


二十三:align-self

-设置单个flex item在cross-axis方向上对齐方式

产品前端构架之模块化 

一:模块组织JavaScript

二:模块
    -封装实现
    -暴露接口
    -声明依赖

三:反模式
    -math.js
        function add(a,b){
            return a+b;
        };
        function sub(a,b){
            return a-b
        }
        -封装无
        -接口结构不明显
    -caculator.js
        var action="add"
        function compute(a,b){
            switch(action){
                case "add":return add(a,b)
                case "sub":return add(a,b)
            }
        }
        -没有依赖声明
        -使用全局状态

四:字面量
-math.js
    var math={
        add:function add(a,b){
            return a+b;
        },
        sub:function sub(a,b){
            return a-b
        }
    }
    -结构性好
    -无访问控制
- caculator.js
    var caculator={
        action:"add",
        compute:function compute(a,b){
            switch(action){
                case "add":return math.add(a,b)
                case "sub":return math.add(a,b)
            }
        }
    }
    -同样没有依赖声明

五:IIFE
    -caculator-1.js
        -var caculator=(function(){
            var action="add"
            return {
                compute: function (a,b){
                    switch(action){
                        case "add":return math.add(a,b)
                        case "sub":return math.add(a,b)
                    }
                }
            }
        })()
    -访问控制
    -无依赖声明

    -caculator-2.js
        -var caculator=(function(m){
            var action="add"
            function compute(a,b){
                switch(action){
                    case "add":
                        return m.add(a,b)
                    case "sub":
                        return m.add(a,b)
                }
            }
            return{
                compute:compute
            }
        })(math)
        -显示了依赖声明
        -仍然污染了全局变量
        -必须手动进行依赖管理

    六:命名空间
        -math.js
            namespace("math",[],function(){
                function add(a,b){return a+b};
                function sub(a,b){return a-b}
                return{
                    add:add,
                    sub:sub
                }
            })
            -无依赖声明

        -caculator.js
            namespace("caculator",["math"].function(m){
                var action = "add"
                function compute(a,b){
                    return m[action](a,b)
                }
                return{
                    compute:compute
                }
            })
            -依赖声明
            -依赖注入</pre>

产品前端构架之模块系统 

一：模块系统
                -依赖管理 加载/分析/注入/初始化
                -决定模块写法

二:Commonjs/module 非浏览器环境的js
    -math.js
        function add(a,b){
            return a+b;
        };
        function sub(a,b){
            return a-b
        }
        exports.add=add
        exports.sub=sub
    -caculator.js
        var math=require("./math");
        function Caculator(container){
            this.add=container.querySelector("xx")
            //do sth
        }
        exports.Caculator=Caculator
        优点
        -依赖管理程度可靠
        -社区活跃，规范接受度高
        -运行时指出，模块定义非常简单
        -文件级的模块作用域隔离
        -可以处理循环依赖
        缺点
        -不是标准组织的规范
        -同步的require，没有考虑浏览器环境

    三:AMD
        -math.js
            define([],function(){
                function add(a,b){
                    return a+b
                }
                function sub(a,b){
                    return a-b
                }
                //接口暴露
                return{
                    add:add,
                    sub:sub
                }
            })

        -caculator.js
        //依赖声明
            define(["./math"],function(math){
                //do sth
                return{
                    Caculator:Caculator
                }
            })
        优点
        -依赖管理成熟可靠
        -社区活跃，规范接受度高
        -转为异步IO环境打造，适合浏览器环境
        -支持类似Commonjs的书写方法
        -通过插件API可支持加载非JS资源
        -成熟的打包构建工具
        缺点
        -模块定义繁琐，需要额外嵌套
        -只是库级别的支持，需要引入额外库
        -无法处理循环依赖
        -无法实现条件加载

    四:systemjs
        -支持加载AMD
        -支持加载Commonjs
        -支持加载ES6
        -支持加载任何资源
    System.paths={
        "app/*":"../es6/*.js"
    }
    System.import("app/caculator").then(function(c){
        new c.Caculator(document.getElementById("caculator"))

CSS布局之规范 

一:规范
                -文件规范
                -注释规范
                -命名规范
                -书写规范
                -其他规范

二:文件规范
    -文件分类
        -通用类
            -reset.css jquery-ul.css
        -业务类
            -alibum toplist
    -文件引入
        -行内样式
            -不推荐
        -外联引入
            -link
        -内联引入
            -style
        -避免在CSS中使用import
    -文件本身
        -文件名
            -由小写字母，数字，中划线组成
        -编码
            -utf-8

三:注释规范
    -块状注释
        -统一缩进
        -在被注释对象之上
    -单行注释
        -文字两端空格
        -在被注释对象之上
    -行内注释
        -文字两端空格
        -在分号之后

四:命名规范
    -分类命名
        -g-header
        -h-header
    -命名格式
        -大小写
        -长度
            -衡量长度和可读性
    -语义化命名
        -以内容语义命名

五:书写规范
    -单行与多行
        -.logo{//do sth}
        -.logo{
            padding://
            color://
        }
        -二选一 遵守之
    -空格与分号
        -空格
            -缩进 2 or 4
            -规则内空格
                -.logo{width: 200px;height: 50px;}
        -分号
            -保留最后一个属性值的分号
    -属性顺序
        -根据属性的重要性按顺序书写
            -显示属性
            -自身属性

  1. /(^\w+)@([a-z0-9]+).([a-z]+$)/gi

先聊聊JS中的正则


JS中的正则有两种风格





  1. var jsfg=new RegExp(“a”,“gi”) //js风格

  2. var perlfg=/a/gi //perlfg风格





JS风格是通过一个正则构造器来构造的 new RegExp(“正则”,”是否区分大小写，全局搜索等等~”)


perlfg风格通过 /正则/是否区分大小写，全局搜索等等~ 这样的形式


okay开始我们第一个正则:不区分大小写全局替换字符串中的a





  1. var str=“abcAbcAABBCC”

  2. var re=/a/gi

  3. var rep=str.replace(re,“替换”)

  4. console.log(rep) //替换bc替换bc替换替换BBCC




 


上面的无论是A还是a全部被替换成了一个字符串“替换”


我们主要注意/a/gi后的gi


g:全局匹配 不加g则匹配一次就不匹配了


i:不区分大小写 不加就只匹配小写a


以上你就完成了第一个正则匹配 是不是很激动？ 别急 咋们继续


 


范例二:查找字符串第一个出现的数字





  1. var str4=‘adsf 23 23 wcs’

  2. var re2=/\d/

  3. console.log(str4.search(re2))//5




okay 火星文出现了 \d 是什么意思？ 先放着咋们先了解下基础


 


先说说正则里面的圆括号，方括号





  1. var str8=“apc aapc ccpc dbpc”

  2. var re6=/[abc]pc/g //a或b或c apc bpc

  3. var re7=/apc|bpc|cpc/g

  4. console.log(str8.match(re6))//apc apc cpc bpc

  5. console.log(str8.match(re7)) //apc apc cpc bpc





match这个函数的功能是以数组的形式返回匹配到的元素.


/apc|bpc|cpc/g 中”|”是什么意思？


我们知道js中用||代表或 那么在js中用单杠代表或


这里的/[abc]pc/g 啥意思呢？


“g”我们知道是全局匹配


/[abc]pc/是shenmegui？


[abc]pc的意思是 a或b或c开头的英文字母后面接一个pc 比如apc,bpc,cpc;可以匹配到这几种


可以得出方括号里的内容都是或的 比如我们要匹配0-9a-z下划线那么可以这样写





  1. var re7=/[0-9a-z]/




而上面这段看起来有点长的正则可以简写为 \w





  1. var re7=/\w/ //等同于上面的正则




再来说说 我们只匹配除0-9以外的东西应该怎么写？





  1. var re8=/[^0-9]/




注意我在方括号的开头加了一个^符号 而这段正则可以这样写





  1. var re8=/\D/ //注意我这里是大写的\D 不是小写的\d




大写小写肯定是有区别的 他们之间区别也很简单


我们知道\D是除0-9以外的东西 那么小写的\d就是 匹配0-9 也就是 /[0-9]/


同理 \w是匹配0-9a-z下划线 那么\W 就是匹配0-9a-z下划线除外的东西 也就是/[^0-9a-z]/


okay学到这里我们再来看看第二个范例


范例二:查找字符串第一个出现的数字





  1. var str4=‘adsf 23 23 wcs’

  2. var re2=/\d/

  3. console.log(str4.search(re2))//5




这里的/\d/并没有加g 也就说只匹配一次 再结合这个\d是什么意思我们很容易得出这个正则表达式就是匹配数字


关于\d \s \b什么意思可以去这个网站进行查询https://msdn.microsoft.com/zh-cn/library/ae5bf541(VS.80).aspx


再来个范例3:替换所有文字为soul





  1. var str9=“abc”

  2. var re12=/./g //.代表任意字符

  3. console.log(str9.replace(re12,“soul”)) //soul soul soul




这里的/./当中的”.”就是代表任意字符 首先上面的案例匹配了a,b,c匹配到了三次 把a替换成了soul 把b替换成了soul 把c替换成了soul


所以会出现三个soul


来点复杂的范例4:过滤HTML标签





  1. var str10=“<p>dasdsa</p>nice <br> test</br>”

  2. var re13=/<[^<>]+>/g

  3. console.log(str10.replace(re13,“”))




okay 先别急着看 我们先了解正则火星文中的量词.说白了就是出现次数



量词:个数

{n}正好出现n次

[1-9]\d{8}只出现八次的数字 第一位为1-9数字

{n,m}最少n次最多m次

[1-9]\d{4,10}

{n,}最少n次，最多不限

+{1,}

?{0,1}最少0次最多一次

*{0,}可以有 最多可以有无限 0次也算 匹配不推荐用


okay 从上表知道{n,m}代表量词 那么上面的范例其实也可以这样改造





  1. var re13=/<[^<>]+>/g

  2. var re13=/<[^<>]{1,}>/g





okay现在知道+号是什么意思了 现在来分析下这个正则





  1. var re13=/<[^<>]+>/g

  2. <[^<>]+>/g 匹配尖括号<[里除尖括号以外的所有字符]字符出现次数最少为一次>





如果不加那个+号会变成什么情况呢？ 这个问题留给大家思考一下


okay 学到这里正则基本上也就入门了. 现在回过头来看看文章开头提到的邮箱匹配正则





  1. /(^\w+)@([a-z0-9]+).([a-z]+$)/gi






这里的()起一个分组的作用 你可以看成数学中的加减乘除所以上例的正则去掉圆括号意思其实是一样的 先分析@前面的正则





  1. (^\w+) //=^\w+





\w是啥意思？匹配任何字类字符，包括下划线。与“[A-Za-z0-9]”等效。 所以邮箱的@前可以有以上的字符并且出现的次数都不限定最少出现一次 但是我为什么要在正则前面加一个^ 这个问题放在后面再说 咋们继续






  1. ([a-z0-9]+). //=[a-z0-9]+.





这里用到了一个.意思为转义字符 我们知道正则中.是匹配任意字符，我们只需要匹配一个单纯的”.”号所以需要用到一个转义字符\ 当然匹配[]也是需要用到转义字符的 [ ]

@gtb123.xxx 这里的gtb123则是匹配到的内容 我们知道邮箱@号后面可以跟数字和字符所以我们要写上可以匹配到数字和字符


现在来看看.后面的正则





  1. ([a-z]+$) //=[a-z]+$





我们知道.后面只能跟英文 比如 .com .cn .net

这里的$是啥意思？ 这里的$代表字符串的结尾 所以这个正则的意思就是 字符串的结尾必须 com net cn 英文结尾的


现在回过头来看看@开头的内容





  1. (^\w+) //=^\w+





这里的^不是代表否,非的意思了 在方括号里面[^0-9]的^代表否，非 那么没在方括号里面的代表的就是字符串开头

所以这个正则的意思就是 字符串必须是以下划线,数字,字母开头的


结合起来这串正则的意思就





  1. /(^\w+)@([a-z0-9]+).([a-z]+$)/gi

  2. 下划线,数字,字母@字母,数字.英文

  3. 比如

  4. test1213_123@AbDd.cn





有了这个正则那么我们应该怎么用来判断呢？ js里有一个test方法





  1. var str10=“sd_blue@etst.com”;

  2. var re13=/(^\w+)@([a-z0-9]+).([a-z]+$)/gi

  3. console.log(re13.test(str10))// true






js中的test方法 如果满足要求则符合true 注意就算字符串中的一部分符合要求也会返回true的.

圆括号不仅仅有分组的意思 还涉及到一个捕获,我们知道match会返回一个数组 举个简单例子





  1. var str19=“2030”;

  2. var re19=/\d+[+-\\/]\d+/

  3. var mat1=str19.match(re19)

  4. console.log(mat1) //[“2030”, index: 0, input: “2030”]




上面是一串匹配数字,符号,数字的正则 看起来比较奇怪 但是还是能够匹配到的. 返回的是一个数组 注意！这时我们加上一个圆括号来分组下





  1. var str19=“2030”;

  2. var re19=/(\d+)([+-\\/])(\d+)/

  3. var mat1=str19.match(re19)

  4. console.log(mat1) //[“2030”, “20”, ““, “30”, index: 0, input: “2030”]




这里返回的数组是[“2030”, “20”, ““, “30”, index: 0, input: “2030”]


0:”2030”


1:”20”


2:”“


3:”30”


可以得出数组[0]为匹配元素 从1开始就得出了分组捕获的内容


上面的正则第一个圆括号分组捕获的是符号前面的数字 第二个圆括号捕获的是符号 第三个圆括号捕获的是符号后面的数字


捕获组还能反向引用，即在表达式中直接使用某个分组的内容。





  1. var str20=“20-20”;

  2. var re20=/(\d+)([+-*\/])(\1)/

  3. var mat2=str20.match(re20)

  4. console.log(mat2) //[“20-20”, “20”, “-“, “20”, index: 0, input: “20-20”]




这里第三个分组 也就是(\1)引用的是第一个分组(\d+)捕获到的内容 同理 如果要引用分组2捕获到的内容那么





  1. var str20=“20-20”;

  2. var re20=/(\d+)([+-*\/])(\2)/

  3. var mat2=str20.match(re20)

  4. console.log(mat2) //[“20-20”, “20”, “-“, “20”, index: 0, input: “20-20”]




改为\2即可 同理 引用3,4,5,… \n即可 别忘记了分组捕获是从1开始的 0为匹配内容


置换前、后两个数的位置





  1. var str21=“178-334”;

  2. var re21=/(\d+)([+-*\/])(\d+)/

  3. var rep3=str21.replace(re21,“$3$2$1”)

  4. console.log(rep3)//334-178




在replace函数中用$1,2,3,n代表分组

产品前端构架之模块化 

一:模块组织JavaScript

二:模块
    -封装实现
    -暴露接口
    -声明依赖

三:反模式
    -math.js
        function add(a,b){
            return a+b;
        };
        function sub(a,b){
            return a-b
        }
        -封装无
        -接口结构不明显
    -caculator.js
        var action="add"
        function compute(a,b){
            switch(action){
                case "add":return add(a,b)
                case "sub":return add(a,b)
            }
        }
        -没有依赖声明
        -使用全局状态

四:字面量
-math.js
    var math={
        add:function add(a,b){
            return a+b;
        },
        sub:function sub(a,b){
            return a-b
        }
    }
    -结构性好
    -无访问控制
- caculator.js
    var caculator={
        action:"add",
        compute:function compute(a,b){
            switch(action){
                case "add":return math.add(a,b)
                case "sub":return math.add(a,b)
            }
        }
    }
    -同样没有依赖声明

五:IIFE
    -caculator-1.js
        -var caculator=(function(){
            var action="add"
            return {
                compute: function (a,b){
                    switch(action){
                        case "add":return math.add(a,b)
                        case "sub":return math.add(a,b)
                    }
                }
            }
        })()
    -访问控制
    -无依赖声明

    -caculator-2.js
        -var caculator=(function(m){
            var action="add"
            function compute(a,b){
                switch(action){
                    case "add":
                        return m.add(a,b)
                    case "sub":
                        return m.add(a,b)
                }
            }
            return{
                compute:compute
            }
        })(math)
        -显示了依赖声明
        -仍然污染了全局变量
        -必须手动进行依赖管理

    六:命名空间
        -math.js
            namespace("math",[],function(){
                function add(a,b){return a+b};
                function sub(a,b){return a-b}
                return{
                    add:add,
                    sub:sub
                }
            })
            -无依赖声明

        -caculator.js
            namespace("caculator",["math"].function(m){
                var action = "add"
                function compute(a,b){
                    return m[action](a,b)
                }
                return{
                    compute:compute
                }
            })
            -依赖声明
            -依赖注入</pre>

产品前端构架之git命令 

一:git help
                -git –help

二:git congfig
    -用户配置
        -git config --global user.name "xxx"
        -git config --global user.email test@xxx.com
    -配置级别
        - --local [默认，高优先级]:只影响本地仓库 .git/config
        - --global [中优先级]:影响到所有当前用户的git仓库 ~/.gitconfig
        - --system [低优先级]:影响到全系统的git仓库 /etc/gitconfig

三:git init
    -git status
        -查看当前仓库信息
    -git init
        -初始化一个git仓库

四:git status
    -内容状态
        -工作目录
        -暂存区
        -提交区
    -了解当前内容状态

五:git add
    -添加文件内容到暂存区（同时文件被跟踪）
    -批量添加
        -git add .  //添加当前目录所有文件

六:忽略文件
    -.gitgnore
        -在添加时忽略匹配的文件
        -仅作用于未跟踪的文件
    -常见的gitgnore
        #logs //注释
        logs
        *.log

七:从暂存区删除git-rm
    -git rm --cached:仅从暂存区删除
    -git rm :从暂存区与工作目录删除
    -git rm $(git ls-files --deleted):删除所有被跟踪，但是在工作目录被删除的文件

八:工作目录与暂存区
    -物品:文件
    -货架:工作目录
    -购物车:暂存区
    -购买:提交内容

    -每类物品只能放置一次的购物车
    -货架和购物车可以出现同种物品
    -货架上的物品可以替换掉购物车物品
    -可以删除物品
    -提交购物车完成购买，生成购买记录

九:git commit
    -根据暂存区内容创建一个提交记录
    -git commit -m ""
    -直接提交
        -git commit -a

十:git log
    -git log --oneline
        -简单log

十一:git中的alias(别名）命令
        -git config alias.shortname &lt;fullcommad&gt;

        -git lg//配置了别名

十二:git diff
    -显示不同的版本差异
    -git diff:
        -工作目录与暂存区的差异
    -git diff --cached
        -暂存区与某次提交差异,默认为HEAD
    -git diff &lt;reference&gt;
        -工作目录与某次提交差异

十三:撤销本地修改

        -将文件内容从暂存区复制到工作目录

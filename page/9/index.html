任务


D3(暗黑3)中任务可以分为主线任务,支线任务,特殊随机任务,悬赏任务(冒险模式)


 


主线任务


一条路走到底的任务.每次做完主线任务都会随机给一件装备.


 


支线任务


支线任务基本上都在FB(副本)里面随机接取的.做完后可以得到一大笔钱和经验以及装备.


看见非主线任务FB都可以进去.因为里面会有一大堆的经验怪与宝箱


 


特殊随机任务


一般都是开诅咒宝箱,诅咒祭坛,沾满鲜血的宝箱,一般来说这种任务都是限时完成的.比如要在60秒杀完一大波兵.在60秒内存活下来 奖励也是挺丰富的.有可能会掉光(传奇装备)


 


悬赏任务


在付费后开启冒险模式,既可以接取悬赏任务,悬赏任务有A1悬赏任务,A2-A5悬赏任务 每章悬赏任务都有5个 全部做完以后会给你一个赫拉迪姆禮包 打开后爆一大堆闪瞎狗眼的装备.


 


 



装备


D3中装备有以下几种


 


白色装


最普通的装备,掉在地上都没人要的装备


 


蓝色装


前期用来保命存活装备.有胜于无.


 


黄色装


中期 后期装备,一般来说加了一大堆的属性.但是很少加特殊的技能


 


暗金色装


加的属性要比黄色装高,但是基本上都加了一个附加技能


比如: 有35%概率对博主造成武器135%的伤害


 


绿色装


这个是套装,一套集齐后加的属性绝对让你得心脏病.


 


打孔


俗话说的好,没有孔的装备不是好装备.


如今的大神都在冒险模式里没日没夜的狂刷打孔器,为了什么？为的就是给装备打个孔.


镶嵌宝石和没镶嵌宝石的装备是两码事


 


宝石


中后期会爆一大堆宝石给你.


宝石一般来说都看自己喜好和装备配合使用,分解装备后宝石会还给你的 NPC不会吞你宝石的 亲

JavaScript深入浅出:OPP(面向对象)概念与继承

OOP在JavaScript中算一大亮点与重点. 废话不多说,只要理解一句话 JavaScript一切皆对象


 


用wordpress举例说明JavaScript继承与对象构成


 


function WP(name,version,author){
        //定义一个函数对象构造器传递三个参数
        //分别是名字，版本，作者
        this.name=name;
        this.version=version;
        this.author=author;
    }
    //在WP原型定义一个公用属性输出WP信息
    WP.prototype.Into=function(){
        console.log(“WP名字是:”+this.name+”WP版本是:”+this.version+”WP作者是”+this.author);
    };
    //定义两个公用属性
    WP.prototype.code=”PHP”;
    WP.prototype.language=[“中文”,”英文”,”日文”,”…”];

首先用函数对象构造器来创建一个函数WP


然后在WP中添加一些属性


 


 


通过Object.create()来继承WP中的属性与方法


 


function Soul(name,version,age,wpname,into){
        //向WP这个函数对象构造器传递name,version
        //传递this是为了指向WP中的this
        WP.call(this,name,version,wpname);
        this.age=age;
        this.into=into;
    }

    Soul.prototype=Object.create(WP.prototype)//用Object.create创建了一个空对象,并且指向是WP的原型
    //继承了WP的原型和方法
    Soul.prototype.constructor=Soul;//constructor指向Soul

    Soul.prototype.Into=function(){
        console.log(‘我的名字:’+this.name+”|我的版本:”+this.version+”|我的年龄:”+this.age+”|我的WP名字:”+this.version);
    }
    //覆盖掉了WP.prototype.Into

    Soul.prototype.intro=function(name){
        console.log(“WP介绍”+this.into+”By:”+name)
    }

constructor是对象中的一个属性意思是谁构造了我


 


 


然后创建Soul实例


 


var new_soul=new Soul(“凌魂”,”2.333”,”17”,”Soul Seeker”,”追逐内心的灵魂,不愿被沉沦在这幻想世界中”);
console.log(new_soul.code);
console.log(“支持语言:”+new_soul.language)//字符串与字符串相加会调用toString方法
new_soul.Into();
new_soul.intro(“凌魂”);

我们先假设 图片设置了自适应 或者说div设置了自适应 如何来控制jquery宽高变化呢？

JavaScript:深入浅出-闭包

要理解闭包，首先必须理解Javascript特殊的变量作用域。


变量的作用域无非就是两种：全局变量和局部变量。


Javascript语言的特殊之处，就在于函数内部可以直接读取全局变量。


详细也可以参考这篇文章


http://www.old.com/soul-get/soul-js/217


 


 


<!doctype html>
<html lang=”en”>
<head>
    <meta charset=”UTF-8”>
    <title>Document</title>

        function outer(){
            var localVal=30;
            return localVal;
            //返回localVal变量
        }
        outer();//30

        //函数可以作为返回值 传参
        function outer_1(){
            var localVal=30;
            return function(){//返回一个匿名函数
                return localVal;
            }
        }
        var func=outer();
        console.log(func)//30 访问到外层的变量

        //常见错误之循环闭包
        function bo(){
            var get_div=document.getElementsByTagName(‘div’)
        for (var i = 0; i < get_div.length; i++) {
            !function(i){
// 叹号后面跟函数!function和加号后面跟函数+function
// 都是跟(function(){})();这个函数是一个意思，都是告诉浏览器自动运行这个匿名函数的，因为!+()这些符号的运算符是最高的，所以会先运行它们后面的函数
                get_div[i].onclick=function(){
                alert(i)
                }
            }(i);
        };
    }

注意事项




针对组件的注意事项


虽然按钮类可以应用到 <a> 和 <button> 元素上，但是，导航和导航条组件只支持 <button> 元素。




链接被作为按钮使用时的注意事项


如果 <a> 元素被作为按钮使用 – 并用于在当前页面触发某些功能 – 而不是用于链接其他页面或链接当前页面中的其他部分，那么，务必为其设置 role=”button” 属性。




跨浏览器展现


我们总结的最佳实践是：强烈建议尽可能使用 <button> 元素来获得在各个浏览器上获得相匹配的绘制效果。


另外，我们还发现了 Firefox <30 版本的浏览器上出现的一个 bug，其表现是：阻止我们为基于 <input> 元素所创建的按钮设置 line-height 属性，这就导致在 Firefox 浏览器上不能完全和其他按钮保持一致的高度。

为 <form> 元素添加 .form-inline 类可使其内容左对齐并且表现为 inline-block 级别的控件。只适用于视口（viewport）至少在 768px 宽度时（视口宽度再小的话就会使表单折叠）。



可能需要手动设置宽度


在 Bootstrap 中，输入框和单选/多选框控件默认被设置为 width: 100%; 宽度。在内联表单，我们将这些元素的宽度设置为 width: auto;，因此，多个控件可以排列在同一行。根据你的布局需求，可能需要一些额外的定制化组件。




一定要添加 label 标签
